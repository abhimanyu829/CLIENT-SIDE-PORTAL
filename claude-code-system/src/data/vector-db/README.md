# Vector DB

## Purpose
pgvector integration for semantic search and AI recommendations.

## How It Works
- Embeddings generated by OpenAI `text-embedding-3-small` (1536 dimensions)
- Stored in `User.embedding` and `Product.embedding` columns (pgvector type)
- Generated asynchronously via BullMQ embedding job
- Queried with cosine similarity for recommendations

## Prompt to Build a Vector Query
```
Build a vector similarity query at src/data/vector-db/[NAME]-search.ts

Purpose: [e.g., find products similar to a user's interests]
Input: [e.g., userId or embedding vector]
Output: [e.g., Product[] sorted by similarity]

Query pattern:
SELECT * FROM "Product"
ORDER BY embedding <=> $1  -- cosine distance
LIMIT $2

Rules:
- Use Prisma $queryRaw for vector queries (Prisma doesn't support <=> operator natively)
- Always exclude already-subscribed products from recommendations
- Cache results in Redis with TTL 10 minutes
- Log similarity scores for eval purposes
- Return only published products (status = PUBLISHED)

Write the complete function with types.
```

## Regenerating Embeddings
```bash
# Queue all products for re-embedding (e.g., after model change)
npx tsx tools/scripts/requeue-embeddings.ts --entity=products
# Queue all users
npx tsx tools/scripts/requeue-embeddings.ts --entity=users
```
